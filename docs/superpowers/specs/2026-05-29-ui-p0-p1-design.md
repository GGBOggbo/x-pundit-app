# 首页与推文页 UI P0-P1 设计

**日期：** 2026-05-29  
**状态：** 已确认并进入实现  
**范围：** 首页评论生成页 + 推文生成页

## 目标

在不扩展功能范围的前提下，提高两条主页面的：

- 首屏价值表达
- 主流程反馈感
- 结果区域信息层级
- 产品一致性与可操作性

## 非目标

- 不改登录、历史记录页面
- 不做品牌重设计
- 不新增后端能力
- 不做复杂动画系统

## P0 问题

### 1. 首屏价值表达偏弱

当前页面虽然信息完整，但用户进入页面后仍然需要自行理解：

- 这是给谁用的
- 输入什么最合适
- 最终会得到什么

### 2. 主流程反馈还不够“像产品”

生成过程中有步骤，但缺少：

- 明确的阶段说明
- 可感知的进展节奏
- 更强的等待引导

### 3. 推文页输入区负担偏大

推文生成页左侧内容较多，历史内容、手动补充、人格、设置都堆在一起，用户需要自己整理优先级。

## P1 问题

### 1. 结果区主次不够清楚

分析摘要、结果卡片、再生成、复制等操作都在，但还没有形成非常清晰的“先看什么、先点什么”的节奏。

### 2. 页面间产品语言不完全统一

评论生成页和推文生成页虽然结构相近，但首屏文案、输入提示、状态反馈、结果标题还不够像同一套产品。

## 设计方案

### A. 强化首屏 Hero 区

首页评论页：

- 在标题下增加更明确的价值描述
- 增加一行“适合场景”提示，降低理解成本
- 把空状态说明从“功能枚举”改成“结果承诺 + 使用路径”

推文生成页：

- 强调“学习你的历史表达，再生成像你写的原创内容”
- 解释“历史内容 + 人格叠加 + 结果生成”的关系

### B. 重组左侧输入区

评论页：

- 保持 3 步结构
- 让 step 标题更像任务提示，而不是单纯标签
- 让输入框和生成设置的层级更清楚

推文页：

- STEP 1 增加“最低要求/推荐量”提示
- 手动补充和历史内容区域做更明显分组
- 把“生成设置”卡片改成更有行动感的总结区

### C. 强化生成过程反馈

- Loading 状态增加阶段解释文案
- 让按钮、提示语、等待区形成一致反馈
- 在无结果、生成中、有结果三种状态下给出更清晰的视觉切换

### D. 提升结果区操作层级

评论页：

- 分析摘要卡更紧凑
- 结果列表前增加“建议先复制/继续润色”的引导

推文页：

- 风格画像摘要更像“你的写作 profile”
- 结果操作栏更突出“换一批 / 复制全部”

### E. 统一产品语言

- 首页和推文页采用一致的 section 语气
- 保持“工具效率 + 产品质感平衡”的方向
- 不靠大换皮，而靠层级、密度、文案和 CTA 清晰度提升体验

## 验证标准

完成后页面应满足：

- 用户进入页面 3 秒内能理解页面用途
- 用户知道输入要求和输出预期
- 生成中和生成后的状态切换更清楚
- 首页和推文页视觉语言更统一
