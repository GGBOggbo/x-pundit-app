# 首页与推文页 10 分前端改版设计

**日期：** 2026-05-29  
**状态：** 待实施  
**范围：** 首页评论生成页 + 推文生成页

## 目标

在不改变核心产品路径的前提下，把当前页面从“好用、有产品感”提升到“更高级、更有记忆点、但仍然高效率”的水平。

这一轮的目标不是新增功能，而是升级：

- 首屏价值表达
- 任务流节奏
- 结果区层级与推荐感
- 页面间统一视觉语言
- 整体品牌记忆点

## 非目标

- 不改登录页和历史记录页
- 不新增后端接口或数据库能力
- 不做商业化相关模块
- 不加入复杂的长动画或重交互特效
- 不重构整体信息架构为全新页面模式

## 当前问题

### 1. 页面可用，但视觉记忆点不足

当前首页和推文页已经清楚、顺手，但整体更接近“稳妥的工具页面”，缺少一眼能记住的首屏表达和品牌气质。

### 2. 左侧仍有较重表单感

虽然已有分步结构，但用户进入页面后仍然主要感知到“在填表”，而不是“在推进一个 AI 工作流”。

### 3. 右侧结果区更像列表，不像价值区

当前结果展示足够实用，但还不够像“已经替你完成了一部分思考”的高价值输出面板。推荐结果、次优结果、下一步动作之间的节奏还不够强。

### 4. 两个主页面还没有形成足够鲜明的统一视觉语言

评论生成页和推文生成页结构类似，但还没有形成真正共享的视觉系统，只是“长得差不多”。

## 设计原则

### 1. 结构不推翻，表现全面升级

保留当前“左输入配置、右结果输出”的工作台结构，不做激进改版。重点放在更强的表现层和更清晰的用户节奏上。

### 2. 工具效率优先，但不能牺牲高级感

页面仍要让用户快速开始、快速出结果；所有视觉升级都必须服务于更快理解和更强信任，而不是仅仅为了装饰。

### 3. 把页面做成“AI 工作台”，不是“普通表单页”

用户应感知到自己正在驱动一个专业工作流：输入、选择策略、等待处理、评估结果、继续优化。

### 4. 用更少但更有力的表达替代堆叠说明

不通过增加大段说明提高理解，而是通过更强的标题、区块层次、胶囊标签、状态引导和结果主次来传达价值。

## 方案概览

### A. 首屏 Hero 升级为“价值场景入口”

首页评论页：

- Hero 不再只是标题与说明
- 增加更强的 headline、副标题和场景胶囊
- 强调“把普通回复整理成更像真人、更能互动的评论”

推文生成页：

- 强调“先学习你的历史表达，再生成像你本人会发的原创内容”
- 让用户一进入就理解“历史表达 + 人格调味 + 输出推文”的关系

视觉上：

- Hero 区要拥有更明显的背景层次和空间感
- 标题层级更大胆
- 芯片化场景提示承担“快速理解”职责

### B. 左侧区域从表单变成任务流

评论页与推文页都保留 3 步结构，但强化为：

- 每一步像一个明确任务
- 每一步有短而有力的引导语
- 当前选择和整体摘要更可感知

具体包括：

- 更清晰的步骤标题
- 更轻量但更有效的 helper 文案
- 增强当前配置的 summary strip
- 让 CTA 更像“启动一次 AI 工作流”

推文页额外处理：

- 让历史内容区和手动补充区形成更明显主次
- 样本数量反馈更清晰
- 生成设置区更有“准备完成，可以开始”的感觉

### C. 右侧结果区升级为“高价值输出区”

首页评论页：

- 先让用户知道“已经生成多少条、建议先看什么”
- 分析摘要与结果列表的过渡更自然
- 第一屏结果更像“推荐结果集合”

推文页：

- 风格画像摘要更像 profile，而不是普通字段展示
- 输出区应让用户理解“这是基于你的历史风格生成的”

两个页面共同点：

- 结果说明更像编辑助手在引导用户决策
- 底部操作区更突出“继续生成”和“整体复制”
- 卡片之间的节奏更强，减少“每条都一样重”的感觉

### D. 建立统一而更鲜明的品牌语言

本轮不换设计系统，但会统一以下元素：

- Hero 区样式语言
- 卡片边框、圆角和背景层次
- 标签、胶囊、辅助说明的语气
- 结果区标题与状态提示风格

目标是让评论页和推文页被看作同一产品中的两个专业工作台，而不是两张功能相近的页面。

## 具体改动范围

### 修改文件

- `app/page.tsx`
- `app/generate-tweets/page.tsx`
- `app/globals.css`

### 可能涉及的小范围调整

- 仅当需要时，允许对现有组件的样式类名做小幅补充
- 不计划重写 `CommentCard` / `TweetCard` 组件逻辑

## 交互与状态要求

### 空状态

- 空状态不应只是“这里以后会有结果”
- 要强调“输入后将获得什么价值”
- 要让用户更愿意开始第一步

### 生成中

- 生成过程要更像系统在工作，而不是单纯 loading
- 状态说明应让等待更有意义

### 结果出现后

- 用户应明确知道先看摘要、再看结果、再决定复制或继续优化
- 首次看到结果时的主次关系要清楚

## 验证标准

完成后应满足以下标准：

- 用户进入首页或推文页后，3 秒内理解页面用途和输出价值
- 左侧配置区域更像任务流，而不是传统表单
- 右侧结果区具备更强的“推荐输出面板”感
- 评论页和推文页拥有明显统一的视觉语言
- 页面整体更像成熟 AI 产品，而不是普通工具页

## 风险与控制

### 风险 1：为了追求高级感而牺牲效率

控制方式：

- 不改变主路径结构
- 不隐藏核心操作
- 不增加多余点击

### 风险 2：视觉升级过多，导致信息密度失衡

控制方式：

- 优先调整层级与节奏，而不是一味增加装饰
- 让每个新增视觉元素都承担信息传达职责

### 风险 3：两个页面风格统一后，丢掉各自差异

控制方式：

- 共享视觉语言
- 保留文案重点和输入路径的差异化表达
